## TL;DR

在 TEC-PLUS **硬连线模式**下，用 **HD0–HD7** 数据开关 + **SWA–SWC** 模式开关，将测试程序灌入 RAM（或微程序模式预灌），再 **SW=000** 单步/连续运行，最后用 **读寄存器** 核对结果。

**上板前必做飞线：** 时序发生器 **T3 → FPGA C10**（详见 §3.1）。**W3** 板内已接 **F4**，无需飞线。

**STO 不是面板上的拨码开关**——它是流程图里的内部两拍状态；你只需在写存/写寄存器时 **按两次单步**，不必找名为 STO 的开关。

## 1. 面板符号对照

| 本文用语 | 板身丝印 | 说明 |
|----------|----------|------|
| 数据开关 | **HD0–HD7** | 8 位输入；**HD7 = IR7（最高位）**，**HD0 = IR0（最低位）** |
| 模式开关 | **SWA、SWB、SWC** | 三位拨码；下文记为 `SWC SWB SWA`（C 为高位） |
| 复位 | **CLR#** | 低有效；按一次让 PC、AR 等回初态 |
| 单步 | **T1 / 中单** 等 | 以箱体按钮丝印为准；每按一次推进一个机器节拍或 W 周期 |
| 控制器选择 | 硬连线 / 微程序 | 必须拨到 **硬连线**（测你的 FPGA 控制器） |

### 1.1 SW 模式表（课设流程图-43）

| SWC | SWB | SWA | 功能 |
|-----|-----|-----|------|
| 0 | 0 | 0 | **取指运行** — 从 RAM 按 PC 执行指令 |
| 0 | 0 | 1 | **写存储器** — 向 RAM 写入一个字节 |
| 0 | 1 | 0 | **读存储器** — 从 RAM 读出一个字节到显示 |
| 0 | 1 | 1 | **读寄存器** — 查看 R0–R3 |
| 1 | 0 | 0 | **写寄存器** — 向 R0–R3 写入初值 |

### 1.2 指令字节与 HD 拨码

```
HD7 HD6 HD5 HD4 | HD3 HD2 | HD1 HD0
   操作码 4 位   |   Rd    |   Rs
```

寄存器编码：**R0=00，R1=01，R2=10，R3=11**（占 IR3–IR0 中对应字段）。

常用机器码：

| 指令 | 十六进制 | HD7…HD0（二进制） |
|------|----------|-------------------|
| `ADD R0,R1` | 0x11 | `0001_0001` |
| `INC R0` | 0x40 | `0100_0000` |
| `STP` | 0xE0 | `1110_0000` |
| `SUB R2,R0` | 0x28 | `0010_1010` |
| `AND R0,R1` | 0x31 | `0011_0001` |
| `LD R1,[R2]` | 0x5A | `0101_0110` |
| `ST R0,[R2]` | 0x62 | `0110_0010` |
| `JC +1` | 0x71 | `0111_0001` |
| `JZ +2` | 0x82 | `1000_0010` |
| `JMP R2` | 0xA0 | `1001_1000` |

## 2. STO 是什么？板子上为什么看不到？

**STO（Store phase）** 是硬布线流程图（图片-43）里 **手动写存储器 / 写寄存器 / 读存储器** 通路的 **内部相位标志**，不是给学生拨的独立开关。

| STO 值 | 写存储器 (SW=001) | 写寄存器 (SW=100) |
|--------|-------------------|-------------------|
| **0（第一拍）** | 数据开关 = **地址** → 控制器发 `LAR` 装入 AR | 数据开关 = **寄存器号** → 选通目标寄存器 |
| **1（第二拍）** | 数据开关 = **数据** → 控制器发 `MEMW`、`ARINC` | 数据开关 = **数据** → 控制器发 `DRW` 写入 |

**操作上：** 在 SW=001 或 SW=100 时，对每一个要写入的「地址+数据」或「寄存器+数据」，**按两次单步**即可，无需寻找 STO 开关。实验箱时序发生器会在连续的单步中自动切换 STO 相位（或由数据通路内部反馈）。

**与当前 RTL 的关系：** 图片-47 **未将 STO 引出到 Spartan-6 输入脚**；`hardwired_ctrl` 内部维护 STO，在 **T3 下降沿** 采样 `SSTO` 置位（地址步 `LAR` 在 **T3 上升沿** 装入 AR，二者错开）。

因此：

- **第一拍**（STO=0）：`SBUS` + `LAR`
- **第二拍**（STO=1）：写存 `MEMW`+`ARINC`；读存 `MBUS`+`ARINC`；写寄存器换 SEL 组合

写寄存器（SW=100）为固定顺序 **R0→R1→R2→R3**，每寄存器占 **W1+W2** 两拍，**不是**在 HD 上拨寄存器号。

|------|------|-----------|------|

说明：

- 早期调试曾把 W3 约束在 **N5** 并飞线，LD/ST 第三拍才正常；后确认 **W3 内部连 F4**，应改 UCF 后 **重新综合/烧录**，并 **去掉 W3→N5 飞线** 再测。
- 课件图片-47 若写 W3=N5，以 **本机 F4 + 板内走线** 为准。
- **T3** 仍必须飞线：AR/PC/IR 等寄存器在板内吃时序 T3，控制器 STO 吃 **C10** 上的 T3。

接线确认（上电前）：

1. 硬连线模式拨码正确；`top.bit` 已烧录（**UCF 中 W3=F4 的新 bit**）。
2. **T3 → C10** 飞线接牢；**不要** 再飞 W3→N5。
3. **CLR#** 复位一次。
4. 核对 HD7=MSB。

### 3.2 其它检查

1. 电源、USB JTAG 接好；ISE IMPACT 已烧录 `top.bit`。
2. 控制器拨码 → **硬连线**。

## 4. 测试用例 A — 双 INC + 停机（推荐首次上板）

**目的：** 只验证取指、INC 译码、STP 停机；**不需要**预先写寄存器。

### 4.1 程序

| 地址 | 机器码 | 指令 | 说明 |
|------|--------|------|------|
| 0 | 0x40 | `INC R0` | R0 ← R0+1 |
| 1 | 0x40 | `INC R0` | 再自增一次 |
| 2 | 0xE0 | `STP` | 停机 |

**预期：** 从 R0=0 出发，运行结束后 **R0 = 2**；遇到 STP 后机器停住（`STOP` 有效）。

### 4.2 操作步骤 — 路径 A（硬连线手动灌程序）

> 适用于写存储器两拍均正常的情况。

| 步骤 | SWC SWB SWA | HD7…HD0 | 操作 |
|------|-------------|---------|------|
| A0 | — | — | **CLR#** 复位 |
| A1 | 0 0 1 | `0000_0000` | 写存 **第 1 拍**：地址 0 → **单步** |
| A2 | 0 0 1 | `0100_0000` | 写存 **第 2 拍**：数据 0x40 → **单步** |
| A3 | 0 0 1 | `0000_0001` | 写存 **第 1 拍**：地址 1 → **单步** |
| A4 | 0 0 1 | `0100_0000` | 写存 **第 2 拍**：数据 0x40 → **单步** |
| A5 | 0 0 1 | `0000_0010` | 写存 **第 1 拍**：地址 2 → **单步** |
| A6 | 0 0 1 | `1110_0000` | 写存 **第 2 拍**：数据 0xE0 → **单步** |
| A7 | 0 1 0 | — | **读存储器** 核对：地址 0→0x40，1→0x40，2→0xE0（读法见 §6） |
| A8 | 0 0 0 | — | 切 **取指运行**；**CLR#** 再复位（PC←0） |
| A9 | 0 0 0 | — | **单步** 若干次，或连续运行，直至 STP 停机 |
| A10 | 0 1 1 | — | **读寄存器 R0**（见 §6） |

### 4.3 预期结果

| 检查项 | 预期 |
|--------|------|
| 读存储器 0/1/2 | 分别为 **40、40、E0**（十六进制） |
| 读寄存器 R0 | **02**（若从 0 起） |
| 停机 | 执行 STP 后不再自动取下一条指令 |

### 4.4 操作步骤 — 路径 B（微程序预灌 + 硬连线执行）

> 当路径 A 写不进 RAM 数据时使用。

| 步骤 | 操作 |
|------|------|
| B0 | 控制器拨到 **微程序**；按实验指导书将上表 3 字节写入地址 0–2 |
| B1 | 拨回 **硬连线**；**CLR#** 复位 |
| B2 | SW ← **000**；单步/运行至 STP |
| B3 | SW ← **011**；读 R0，预期 **02** |

## 5. 测试用例 B — ADD（需寄存器初值）

**目的：** 验证 ALU 加法通路与 `DRW`。

### 5.1 程序与初值

| 项目 | 值 |
|------|-----|
| 初值 | R0 = **1**，R1 = **2**（手动写入） |
| 地址 0 | 0x11 — `ADD R0,R1` |
| 地址 1 | 0xE0 — `STP` |

**预期：** 运行结束后 **R0 = 3**。

### 5.2 写寄存器操作（SW = 100）

写寄存器按 **R0→R1→R2→R3 固定顺序**，每个寄存器 **W1、W2 各单步一次**（共 4 组）。HD 上只拨 **数据字节**，不拨寄存器号。

**写 R0=1、R1=2（仅前两组示例）：**

| 顺序 | 拍 | HD7…HD0 | 操作 |
|------|-----|---------|------|
| R0 | W1 | `0000_0101` | 数据 5 → **单步** |
| R0 | W2 | （无关） | **单步** |
| R1 | W1 | `0000_0011` | 数据 3 → **单步** |
| R1 | W2 | （无关） | **单步** |

完整写 R0–R3 需继续 W1/W2 写 R2、R3（见 **§6.2** 初值表）。

### 5.3 写程序并运行

按 **§4.2** 相同流程，仅向地址 0 写 **0x11**、地址 1 写 **0xE0**（或路径 B 微程序预灌）。

| 步骤 | SW | 操作 |
|------|-----|------|
| 1 | 100 | 写入 R0=1、R1=2（§5.2） |
| 2 | 001 | 地址 0←0x11，地址 1←0xE0 |
| 3 | 000 | **CLR#** → 运行 |
| 4 | 011 | 读 R0 → 预期 **03** |

## 6. 测试用例 C — 已实现指令集综合验收（RUN 模式）

**目的：** 在一条小程序中覆盖当前 bitstream **已实现** 的 RUN 指令（ADD / SUB / AND / INC / LD / ST / JC / JZ / JMP / STP），并用手动模式灌入初值与程序后核对寄存器与 RAM。

**范围说明（当前 RTL）：**

| 已实现 | 未实现（测前跳过） |
|--------|-------------------|
| ADD, SUB, AND, INC | OUT (1010) |
| LD, ST | IRET (1011) |
| JC, JZ, JMP | DI (1100), EI (1101) |
| STP | |

### 6.1 主程序（地址 0x00–0x0C）

主程序覆盖 ADD / SUB / AND / INC / LD / ST / JZ / JC / STP；**JMP 单独在 §6.6 补测**，避免与 JZ 跳过区共用地址。

| 地址 | 机器码 | 指令 | 覆盖 | 运行后效果（相对 §6.2 初值） |
|------|--------|------|------|------------------------------|
| 0x00 | 0x11 | `ADD R0,R1` | ADD | R0 ← 5+3 = **0x08** |
| 0x01 | 0x5A | `LD R1,[R2]` | LD | R1 ← M[0x10] = **0xAA** |
| 0x02 | 0x62 | `ST R0,[R2]` | ST | M[0x10] ← **0x08** |
| 0x03 | 0x31 | `AND R0,R1` | AND | R0 ← 0x08 & 0xAA = **0x08** |
| 0x04 | 0xF0 | `INC R3` | INC | R3 ← **0x01** |
| 0x05 | 0x28 | `SUB R2,R0` | SUB | R2 ← 0x10−8 = **0x08** |
| 0x06 | 0x20 | `SUB R0,R0` | SUB / 置 Z | R0 ← **0x00**，Z=1 |
| 0x07 | 0x82 | `JZ +2` | JZ | Z=1 → 跳过 0x08、0x09 |
| 0x08 | 0xF0 | `INC R3` | — | JZ 跳过，不执行 |
| 0x09 | 0xF0 | `INC R3` | — | JZ 跳过，不执行 |
| 0x0A | 0x71 | `JC +1` | JC | C=0 → 不跳，顺序执行 |
| 0x0B | 0xF0 | `INC R3` | INC | R3 ← **0x02** |
| 0x0C | 0xE0 | `STP` | STP | 停机 |

### 6.2 运行前初值（手动模式灌入）

| 项目 | 值 | 灌入方式 |
|------|-----|----------|
| R0 | 0x05 | SW=100，第 1 组 W1 写 5 |
| R1 | 0x03 | SW=100，第 2 组 W1 写 3 |
| R2 | 0x10 | SW=100，第 3 组 W1 写 0x10 |
| R3 | 0x00 | SW=100，第 4 组 W1 写 0 |
| M[0x10] | 0xAA | SW=001，地址 0x10 → 数据 0xAA |
| 程序 0x00–0x0C | 见 §6.1 | SW=001 逐字节写存 |

### 6.3 操作步骤

| 步骤 | SW | 操作 |
|------|-----|------|
| C0 | — | **CLR#** |
| C1 | 100 | 按 §6.2 写 R0–R3（8 次单步：每组 W1/W2） |
| C2 | 001 | 写 M[0x10]=0xAA；再写程序字节 0x00–0x0C |
| C3 | 010 | 可选：读 M[0x10] 核对为 AA |
| C4 | 000 | **CLR#**（PC←0）→ 单步/连续运行至 **STP** |
| C5 | 011 | 读 R0–R3 |
| C6 | 010 | 读 M[0x10] 核对 |

### 6.4 预期结果

| 检查项 | 预期 |
|--------|------|
| R0 | **0x00**（SUB R0,R0） |
| R1 | **0xAA**（LD） |
| R2 | **0x08**（0x10−0x08） |
| R3 | **0x02**（一次 INC 成功 + JZ 跳过两次 INC + JC 后一次 INC） |
| M[0x10] | **0x08**（ST 覆盖原 AA） |
| 停机 | 执行 STP 后 STOP 有效 |

### 6.5 指令与 W 周期对照（本程序）

| 指令类型 | W1 | W2 | W3 |
|----------|----|----|-----|
| ADD / SUB / AND / INC / JC / JZ / JMP / STP | 取指 | 执行 | — |
| LD / ST | 取指 | LAR+LONG | DRW+MBUS 或 MEMW |

本程序 13 条指令：约 **28** 个 W 拍（11 条双拍 + LD/ST 各多 1 个 W3），遇 STP 停机。

### 6.6 JMP 补测（可选）

与主程序分开，避免修改 R2 影响 LD/ST。

| 地址 | 机器码 | 说明 |
|------|--------|------|
| 0x20 | 0xA0 | `JMP R2` |
| 0x21 | 0xE0 | `STP` |

**步骤：** 写寄存器 **R2=0x21**；写存 0x20←0xA0、0x21←0xE0；**CLR#** 后用手动方式令 **PC=0x20**（或把该 2 字节接在 0x0D 之后并从 PC=0x0D 运行）；单步/运行后应 **STP 停机**。

## 7. 读回与核对

### 7.1 读存储器（SW = 010）

对每个字节：**两拍单步**。

| 拍 | HD7…HD0 | 作用 |
|----|---------|------|
| 1 | 目标 **地址** | `LAR` |
| 2 | （无关） | `MBUS` 读出；在 **数据指示灯 / 数码管** 上查看 |

### 7.2 读寄存器（SW = 011）

固定 **两拍**：W1 选 R0/R1 组，W2 选 R2/R3 组（与写寄存器顺序一致，**不在 HD 上拨寄存器号**）。

| 拍 | 操作 |
|----|------|
| 1 | **单步**（W1）— 查看 R0、R1 |
| 2 | **单步**（W2）— 查看 R2、R3 |

核对 **§6.4** 时：R0/R1 在拍 1 后读，R2/R3 在拍 2 后读（以箱体显示为准）。

## 8. 单步运行时的 W 周期（便于对波形）

SW=000 时，每条指令典型时序：

| 周期 | 控制器行为（摘要） |
|------|-------------------|
| **W1** | `LIR` + `PCINC`（取指） |
| **W2** | 按操作码执行（INC/ADD/STP 等） |
| **W3** | 仅 LD/ST 需要；本测试用例无 |

**测试用例 A** 共 3 条指令，约 **6 个 W 周期**（每条 2 拍）后 STP 停机。  
**测试用例 C** 见 §6.5（约 28 个 W 拍，含 LD/ST 的 W3）。

## 9. 故障排查

| 现象 | 可能原因 | 建议 |
|------|----------|------|
| LD/ST 进入 W3 无控制字 | UCF 中 W3 引脚错误（应用 **F4** 非 N5） | 改 `tecplus.ucf` → 重综合烧录；去掉 W3→N5 飞线 |
| 写存地址装不进 | STO 在 T3↑ 与 LAR 冲突 | 已修：STO 改 **T3 下降沿** 更新；并确认 **T3 → C10** 飞线 |
| 灌数据时显示 AR 而非开关 | 地址步失败或 ARINC 同拍 | 先确认 §6 写存两拍正常 |
| R0 不对 | 未复位 PC；或 HD 高低位反了 | CLR# 后再跑；核对 HD7=MSB |
| 不停机 | STP 未译码 | 对照仿真；查 SW 是否为 000 |
| JZ/JC 行为异常 | Z/C 标志未建立 | 先跑 SUB R0,R0 等置标志指令 |
| 与微程序结果不一致 | 硬连线控制字偏差 | 同程序在微程序模式跑一遍对比 |
